# ADR-0006: 제품명 확정 — Neulsang

- 날짜: 2026-07-04 / 상태: 승인

## 맥락
PRD(`docs/prd.md`)는 제품명으로 "DevRecall"을 추천했으나 확정하지 않았고, 저장소 폴더명(`neulsang-dictionay`)과도 달랐다. 코드·문서·모듈 경로에 이름이 박히기 전에 하나로 확정해야 이후 일괄 치환 비용을 최소화할 수 있다 (CLAUDE.md 제품명 규칙).

## 결정
제품명을 **Neulsang**(한글 "늘상")으로 확정한다. 저장소 폴더명(`neulsang`)과 일치시킨다.

- 코드·문서 전체에서 기존 "DevRecall" 표기를 "Neulsang"으로 일괄 치환한다.
- 영문 표기는 `Neulsang`(고유명사, 첫 글자 대문자), 한글 병기는 "늘상".
- 컴포넌트·바이너리 이름(`desktopd` 등)은 제품명과 별개이므로 바꾸지 않는다.

## 근거
- 저장소 폴더명과 일치 → 이름 불일치로 인한 혼란 제거
- "늘상(늘, 항상)"은 "업무 중 늘 곁에 두고 쓰는 도구"라는 상주형(트레이 앱) 제품 성격과 맞음
- DevRecall은 영어 조어라 명확하지만, 사용자가 한국어 화자용 개인 프로젝트로 폴더명 기반 이름을 선호

## 미채택 대안
- **DevRecall 유지**: PRD 추천안. 의미는 명확하나 폴더명과 불일치가 남음.
- **둘 다 보류**: 코드가 쌓일수록 치환 비용이 커져 초기에 확정하는 편이 유리하다고 판단.

## 결과·트레이드오프
- PRD 원본을 포함해 문서의 "DevRecall" 표기를 "Neulsang"으로 치환 (이 ADR이 PRD의 이름 추천을 대체함)
- go module path·package명은 코드 착수(백로그 #1) 시점에 `neulsang` 기준으로 정한다
- 저장소 폴더명의 철자 오타(`dictionay`)는 제품명과 무관한 별개 항목으로, 사용자가 별도 처리
