대상: $ARGUMENTS (파일 경로 또는 디렉토리. 비어 있으면 최근 변경분)

세 엔진이 서로 다른 관점으로 보게 하는 것이 목적이다. 순서:

1. **codex 리뷰** (구현 타당성·엣지케이스 관점):
   ```
   codex exec --sandbox read-only \
     "다음 대상을 비판적으로 리뷰하라. 칭찬은 생략하고 문제점·리스크·누락만 지적하라. 각 지적에 심각도(high/med/low)를 붙여라. 무엇이 부족한지와 무엇이 과한지를 모두 답하라. 대상: <대상>" \
     -o /tmp/tri-review-codex.txt
   ```
2. **agy 리뷰** (아키텍처·확장성·대용량 컨텍스트 관점):
   ```
   agy -p "다음 대상을 비판적으로 리뷰하라. 동의보다 반박에 집중하고, 각 지적에 심각도(high/med/low)를 붙여라. 무엇이 부족한지와 무엇이 과한지를 모두 답하라. 대상: <대상>" --print-timeout 10m < /dev/null
   ```
   1과 2는 병렬 실행 가능.
3. **Claude 종합**: 두 리뷰를 읽고 각 지적을 검증한다 — 실제 파일을 확인해 사실이 아닌 지적은 기각하고 사유를 남긴다. 결과를 표로 정리: (지적 | 출처 | 심각도 | 판정: 수용/기각/보류 | 조치).
4. 수용된 항목 중 즉시 고칠 수 있는 것은 사용자 확인 후 수정한다.

주의: 두 CLI의 지적을 그대로 믿지 말 것. 검증 없는 수용 금지.
